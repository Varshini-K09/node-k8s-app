Docker Installation

1. Install Docker

Install Docker using DEB packages

docker deb files download latest docker-ce, docker-ce-cli, containerd.io deb files

copy all files into dockersw folder

Step 0: Go to Docker DEB directory

cd dockersw

Step 1: Install Docker using DEB packages

sudo dpkg -i *.deb
sudo apt --fix-broken install

Step 2: Enable Docker to start on boot

sudo systemctl enable --now docker

Step 3: Verify Docker Installation

docker --version

Check Docker CLI(Client) installation

Step 4: Check Docker service status

sudo systemctl status docker

Check Docker daemon is running

Step 5: Add current user to docker group

sudo usermod -aG docker $USER

After adding the user to the group, you can run Docker without sudo:

Step 6: Refresh shell session

Log out and log back in to apply group changes.

Step 7: Run test container

docker run hello-world

It verifies that the Docker engine works and that containers can be executed without issues.

Step 8: Verify Docker is working

docker ps

Shows the list of currently running containers on your system.

Explanation:
This confirms Docker is running correctly and the Docker daemon is responsive.

Docker Images

2: Pull Docker Image

Step 1: Pull official Node.js 24 LTS Linux image

docker pull node:24-bookworm-slim

Step 2: Run Node inside container

docker run -it node:24-bookworm-slim node -v

Expected output: 

v24.x.x
Concept:
Node.js is running inside a Linux container without installing Node on the host system.

Step 3: Listing Docker Images

To view all downloaded images on your system, run:

docker images

This displays the image repository, tag, image ID, and size.

3: Create Node.js Application

math.js

function add(a, b) {
  return a + b;
}

module.exports = add;

app.js

const add = require('./math');

console.log("Sum:", add(2, 3));

math.test.js

const add = require('./math');

test('adds 2 + 3 to equal 5', () => {
  expect(add(2, 3)).toBe(5);
});

package.json

{
  "name": "my-node-app",
  "version": "1.0.0",
  "private": true,
  "type": "commonjs",
  "scripts": {
    "start": "node app.js",
    "test": "jest"
  },
  "devDependencies": {
    "jest": "^29.7.0"
  }
}

4: Dockerfile Creation

Dockerfile

FROM node:24-bookworm-slim

WORKDIR /app

# Copy only package files first
COPY package*.json ./

# Install all dependencies (including jest)
RUN npm install

# Copy application source code
COPY . .

CMD ["npm", "start"]
Important:
The FROM instruction defines the Linux OS and Node.js runtime. Without it, the Docker image cannot be built.

5: Build Docker Image

sudo docker build -t my-node-app .

This command creates a Docker image named my-node-app.

6: Verify the Docker Image

List all Docker images available on your system:

docker images

You will see my-node-app listed in the images table.

7: Running a Container from the Image

Run your Docker image:

sudo docker run --rm my-node-app

This executes the CMD instruction defined in the Dockerfile (npm start).

To start an interactive Node.js container shell:

sudo docker run -it node:24-bookworm-slim /bin/bash

Expected output: 

Sum: 5
Note:
The --rm option automatically removes the container after execution.

8: Run Tests Inside Docker Container

sudo docker run --rm my-node-app npm test

Expected result: 

PASS ./math.test.js
✓ adds 2 + 3 to equal 5
Key Learning:
The same Docker image can be used to:
  • Run the application
  • Run automated tests
No Node.js or Jest installation is required on the host system.

Introduction to Docker Containers

Docker containers enable applications to run in isolated and consistent environments across different systems. They help build, test, and deploy applications faster by eliminating compatibility issues.

What is a Docker Container?

A Docker container is a lightweight, standalone, executable package that includes:

  • Application code
  • System tools
  • Runtime libraries
  • Configurations

Unlike VMs, containers share the host OS kernel, making them faster and more efficient.

Why Use Docker Containers?

  • Portability: Runs anywhere Docker is installed.
  • Consistency: Eliminates environment-specific issues.
  • Efficiency: Uses fewer resources and starts faster.
  • Scalability: Easily replicate and manage containers.

Using Nginx as Example

We will use the Nginx web server image from Docker Hub to demonstrate container operations.

Pulling an Nginx Image

Download the Nginx image from Docker Hub:

docker pull nginx

Running an Nginx Container

docker run -d -p 8080:80 --name webserver nginx

Explanation:

  • -d: Detached mode
  • -p 8080:80: Maps host 8080 → container 80
  • --name webserver: Custom container name

Open http://localhost:8080 to see Nginx running.

Serving Your Own Files

Step 1: Create HTML Files in your home directory

mkdir my-site
echo "
Hello from Nginx Container
" > my-site/index.html

Step 2:Remove old webserver container

docker rm -f webserver

Step 3: Serve Files Using Volume Mount

docker run -d -p 8080:80 --name webserver -v $(pwd)/my-site:/usr/share/nginx/html nginx
  • -v $(pwd)/my-site:/usr/share/nginx/html maps your local folder to Nginx’s web root.

Your custom HTML is now served at http://localhost:8080.(do not use https)

What is a Docker Volume?

A volume allows data sharing between host and container.

  • Use -v host_directory:container_directory
  • Useful for serving files or persisting data

Container Lifecycle Commands

docker ps           # Running containers
docker ps -a       # All containers
docker stop webserver
docker start webserver
docker restart webserver

Access & Monitoring Commands

docker exec -it webserver bash
docker logs webserver
docker logs -f webserver
docker inspect webserver

Cleanup Commands

docker rm webserver
docker rm -f webserver
docker container prune

Conclusion

Docker containers provide an efficient, portable, and isolated environment for running applications.

Key Takeaways:
  • Containers ensure consistent deployment across systems
  • Easy to manage using commands like run, stop, start, rm
  • Supports live logs, shell access, and configuration inspection
  • Volumes help share and persist data

Mastering containers is essential for modern cloud-native application development.

Understanding Dockerfiles and Building Applications

A Dockerfile is a text file named Dockerfile (without extension) that contains instructions to build Docker images. Each instruction defines a specific step such as choosing a base image, installing packages, copying files, or defining how the container should start. Mastering Dockerfiles enables reproducible, automated, and efficient application deployments.

Why Use a Dockerfile?

A Dockerfile automates image creation and eliminates manual environment setup. Key benefits:

  • Consistency across development, testing, and production
  • Automation of all installation & configuration steps
  • Version control for environments
  • Portability—run anywhere using Docker

Dockerfile Syntax and Instructions

InstructionDescription
FROMSpecifies base image
RUNExecute commands during image build
COPYCopy files into image
ADDCopy files (supports URLs, archives)
CMDDefault command when container starts
ENTRYPOINTDefines executable container entry
WORKDIRSets working directory
ENVDefine environment variables
EXPOSEDocument container port

Below is a full Node.js web application demonstrating how Dockerfile works inside a real project.

package.json

{
  "name": "my-node-app",
  "version": "1.0.0",
  "description": "Sample Node.js app to demonstrate Dockerfile usage",
  "type": "commonjs",
  "scripts": {
    "start": "node index.js",
    "test": "jest"
  },
  "dependencies": {
    "express": "^4.19.2"
  },
  "devDependencies": {
    "jest": "^29.7.0"
  }
}

index.js


const express = require("express");
const add = require("./routes/math");

const app = express();

app.get("/", (req, res) => {
  res.send("Welcome to Node.js Docker Demo App!");
});

app.get("/add", (req, res) => {
  const result = add(5, 7);
  res.json({ sum: result });
});

app.listen(3000, () => {
  console.log("Server running on port 3000");
});

routes/math.js


function add(a, b) {
  return a + b;
}
module.exports = add;

tests/math.test.js


const add = require("../routes/math");

test("adds 5 + 7 to equal 12", () => {
  expect(add(5, 7)).toBe(12);
});

Dockerfile (Node 24 Bookworm Slim)


FROM node:24-bookworm-slim
WORKDIR /app
COPY package.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]

Build and Run

Build image:

docker build -t my-node-app .

Run container:

docker run -p 3000:3000 my-node-app

Run tests inside Docker:

docker run --rm my-node-app npm test
